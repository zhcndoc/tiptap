## 许可证和源代码

Tiptap [UI 组件](/ui-components/components/overview) 遵循与其构建的 [编辑器扩展](/editor/extensions/overview) 相同的许可证。
- 如果扩展是开源的（例如 Bold、Heading 或 Image），那么 UI 组件也是 MIT 许可证的。
- 你可以自由地查看源代码、复制它，并根据自己的需要进行定制。

某些组件是为付费 **Tiptap 功能** 构建的，例如评论或版本历史。

- 这些组件 **不是** 开源的。
- 大多数包含依赖于 Tiptap 后端服务的前端逻辑。
- 使用它们需要 Tiptap Cloud 订阅或试用。

无论类型（免费或云连接），你都可以检查源代码并根据你的用例进行调整。

## 设计默认
组件的设计旨在让人感觉简单并融入你的设计。它们采用中性色彩：最少的颜色、基本的间距，并且没有强烈的视觉观点。你可以按原样使用它们或进行定制以匹配你的品牌。

所有组件的设计一致且协调，因此如果你同时使用多个组件，它们将很好地匹配，而无需额外的工作。

- 不需要复杂的覆盖或 `!important` 黑客
- 只需直接编辑代码，因为它旨在被分支或扩展

## 组件和原语
为了在你的 Tiptap 编辑器中构建出色的编辑体验，你可以集成 UI 元素。库提供了不同类型的构建模块：**组件**、**节点组件**和 **原语**。

### 组件

现成的、特定功能的 React 组件，例如标题下拉菜单。每个 UI 组件通常与 Tiptap 扩展配对（例如，**ListButton** 依赖于与列表相关的扩展），并使用 React hooks 与编辑器的状态进行交互。

- **与扩展配合使用**：要使用组件（例如 `HeadingDropdownMenu`），请确保在编辑器中启用对应的 Tiptap 扩展（如 `@tiptap/extension-heading`）。
- **利用 hooks**：许多组件内部使用 hooks（如 `useTiptapEditor` 或 `useFloatingElement`）以与编辑器状态同步、处理定位或管理键盘导航。你也可以在自己的自定义组件中使用这些 hooks。
- **易于定制**：默认情况下，这些组件具有最少的样式，因此可以融入你的设计。你可以完全覆盖或扩展它们。每个组件的源代码都是公开的，可以直接编辑，没有 `!important` 黑客。

### 原语

低级 UI 元素，如 `Button`、`DropdownMenu`、`Popover` 和 `Spacer`。它们是许多 UI 组件的基础，处理可访问性和基本行为。你可以将这些原语混合搭配，以创建自己的编辑器控件或进一步自定义现有组件。
