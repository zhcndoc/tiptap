## 1. 从 AI 服务器读取文档内容

首先以简写格式读取文档。调用：

- `POST /v3/ai/toolkit/read/read-document`

```ts
import { getAuthHeaders } from '@/lib/server-ai-toolkit/get-auth-headers'

const readResponse = await fetch(`${apiBaseUrl}/toolkit/read/read-document`, {
  method: 'POST',
  headers: getAuthHeaders(),
  body: JSON.stringify({
    editorContext,
    format: 'shorthand',
    reviewOptions: {
      mode: 'disabled',
    },
    experimental_documentOptions: {
      documentId,
      userId: 'ai-assistant',
    },
  }),
})

const readResult = await readResponse.json()
```

## 2. 生成校对器操作

从 AI 服务器获取工作流定义，将其与编辑器上下文提示合并，然后
要求模型返回校对器操作。

```ts
const workflowResponse = await fetch(`${apiBaseUrl}/toolkit/workflows/proofreader`, {
  method: 'POST',
  headers: getAuthHeaders(),
  body: JSON.stringify({
    format: 'shorthand',
  }),
})

const workflow = await workflowResponse.json()

const toolkitResponse = await fetch(`${apiBaseUrl}/toolkit/tools`, {
  method: 'POST',
  headers: getAuthHeaders(),
  body: JSON.stringify({
    editorContext,
  }),
})

const { prompt: toolkitPrompt } = await toolkitResponse.json()

const result = streamText({
  model,
  system: `${workflow.systemPrompt}\n\n${toolkitPrompt}`,
  prompt: JSON.stringify({
    content: readResult.output.content,
    task: 'Correct all grammar and spelling mistakes',
  }),
  output: Output.object({ schema: z.fromJSONSchema(workflow.outputSchema) }),
})

const output = await result.output
```

## 3. 执行工作流

直接执行该工作流，以将修正应用到文档。调用：

- `POST /v3/ai/toolkit/execute-workflow/proofreader`

```ts
const executeResponse = await fetch(`${apiBaseUrl}/toolkit/execute-workflow/proofreader`, {
  method: 'POST',
  headers: getAuthHeaders(),
  body: JSON.stringify({
    editorContext,
    format: 'shorthand',
    input: output,
    reviewOptions: {
      mode: 'disabled',
    },
    experimental_documentOptions: {
      documentId,
      userId: 'ai-assistant',
    },
  }),
})

const executeResult = await executeResponse.json()
```

## 4. 从编辑器 UI 触发工作流

客户端将文档 ID 和编辑器上下文发送到工作流端点。

```tsx
const runProofreader = async () => {
  const response = await fetch('/api/server-proofreader-workflow', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
    },
    body: JSON.stringify({
      documentId,
      editorContext: getEditorContext(editor),
    }),
  })
}
```

## 跟踪更改（Tracked changes）

将此工作流与[跟踪更改](/tracked-changes/getting-started/overview)扩展集成，以便在 AI 编辑文档后显示审阅界面，并允许用户接受和拒绝更改。

在 `POST /v3/ai/toolkit/execute-workflow/proofreader` 请求中，配置 `reviewOptions` 参数：

```ts
reviewOptions: {
  mode: 'trackedChanges',
  trackedChangesOptions: {
    userId: 'ai-assistant',
  },
}
```

查看 [AI Toolkit demos](https://github.com/ueberdosis/ai-toolkit-demos) 以了解如何将 Server AI Toolkit 工作流与跟踪更改集成。

你可以让 AI 为每一处修正提供理由。每个理由都会成为一个注释线程，并通过 [Comments](/comments/getting-started/overview) 扩展与其跟踪更改关联。

### 带有 `operationMeta` 的工作流定义

在获取校对器工作流定义时，传入 `operationMeta` 以要求每个操作都包含一个 `meta` 字段。你提供的字符串用于描述 `meta` 字段中应该包含的内容。

```ts
const workflowResponse = await fetch(`${apiBaseUrl}/toolkit/workflows/proofreader`, {
  method: 'POST',
  headers: getAuthHeaders(),
  body: JSON.stringify({
    format: 'shorthand',
    operationMeta:
      '本次修正的具体原因（例如，“拼写：excelent → excellent”或“主谓一致”。）',
  }),
})
```

返回的 `outputSchema` 现在要求每个操作都包含一个 `meta` 字段，而 `systemPrompt` 会指示 AI 填充它。

### 使用注释执行工作流

执行工作流时，除了 `reviewOptions` 之外，还要传入 `experimental_commentsOptions`：

```ts
const executeResponse = await fetch(`${apiBaseUrl}/toolkit/execute-workflow/proofreader`, {
  method: 'POST',
  headers: getAuthHeaders(),
  body: JSON.stringify({
    editorContext,
    format: 'shorthand',
    input: output,
    reviewOptions: {
      mode: 'trackedChanges',
      trackedChangesOptions: {
        userId: 'ai-assistant',
      },
    },
    experimental_commentsOptions: {
      threadData: { userName: 'Tiptap AI' },
      commentData: { userName: 'Tiptap AI' },
    },
    experimental_documentOptions: {
      documentId,
      userId: 'ai-assistant',
    },
  }),
})
```

操作中每个非空的 `meta` 字段都会成为一个注释线程，并与其跟踪更改关联。该理由会作为线程的第一条评论内容存储，并作为线程数据中的 `suggestionReason` 存储。

### 客户端设置

在客户端，添加带有 `TiptapCollabProvider` 的 `CommentsKit` 扩展：

```tsx
import { CommentsKit } from '@tiptap-pro/extension-comments'
import { TrackedChanges } from '@tiptap-pro/extension-tracked-changes'
import { ServerAiToolkit } from '@tiptap/server-ai-toolkit'

const editor = useEditor({
  extensions: [
    StarterKit.configure({ undoRedo: false }),
    Collaboration.configure({ document: doc }),
    TrackedChanges.configure({ enabled: false }),
    ServerAiToolkit,
    CommentsKit.configure({
      provider, // 你的 TiptapCollabProvider 实例
    }),
  ],
})
```

用户可以在评论侧边栏中查看每个跟踪更改，并阅读 AI 给出的修正理由。
