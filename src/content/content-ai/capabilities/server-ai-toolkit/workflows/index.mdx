每个工作流都遵循相同的三步流程：

1. 使用工作流读取端点读取相关文档状态。
2. 让你的模型生成结构化的工作流输出。
3. 在 Server AI Toolkit API 上执行工作流。

## 工作流 vs. AI 代理

[AI 代理](/content-ai/capabilities/server-ai-toolkit/agents) 可以从多个工具中选择，并
动态地将它们组合起来。工作流更简单：模型为一个已知的服务端
操作生成输入。

当任务固定且可预测时使用工作流。当模型需要在
多个文档操作之间进行选择时，使用代理。
