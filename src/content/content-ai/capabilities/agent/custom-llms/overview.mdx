和其他 Tiptap AI 扩展一样，AI Agent 有两种使用方式：

- 使用 Tiptap Cloud：开箱即用，无需服务器设置。
- 使用您自己的后端：让您对 AI 模型和工具拥有更多控制权。

本指南说明了何时使用每种选项。

## Tiptap Cloud

使用 Tiptap Cloud 的 AI Agent 是最快的入门方式。它免去了构建和维护自己服务器的麻烦。

Tiptap Cloud 可高度自定义：

- 通过编辑[系统提示](/content-ai/capabilities/agent/configure/system-prompt)来自定义 AI Agent 的行为。定义多个 AI Agent，拥有各自的规则和个性。
- 向用户消息[提供上下文](/content-ai/capabilities/agent/features/context)。
- [自定义 AI 模型](/content-ai/capabilities/agent/use-with-content-ai-cloud#customize-the-openai-model)。Tiptap Cloud 仅支持 OpenAI 模型。

对于初学者，建议使用 Tiptap Cloud 的 AI Agent。开始使用请[参考本指南](/content-ai/capabilities/agent/use-with-content-ai-cloud)。

## 自定义后端

<Callout title="初学者提示" variant="info">
  第一次使用 AI Agent？**请先通过 Tiptap Cloud 进行设置**，当需要更多功能时，再迁移到自定义后端。迁移过程无缝衔接，无需修改原有客户端代码。
</Callout>

自定义后端能让您获得更多控制权，但搭建和维护工作量也更大。适用于以下情况：

- 需要使用 Tiptap Cloud 不支持的 AI 模型。
- 需要定义[自定义工具](/content-ai/capabilities/agent/custom-llms/tools)，例如：
  - 与编辑器交互的工具。例如自动格式化文档的工具。
  - 数据检索工具。例如查询天气预报的工具。
- 将 AI Agent 扩展集成到您托管在自身后端的现有 AI Agent 中。

开始使用自定义后端，请[参阅此指南](/content-ai/capabilities/agent/custom-llms/get-started)。
