配置审核工作流程，使用户能够更好地控制 AI 生成的内容。AI Agent 扩展与 [AI 更改](/content-ai/capabilities/changes/overview) 和 [AI 建议](/content-ai/capabilities/suggestion/overview) 扩展集成，帮助用户批准、拒绝和修改 AI 生成的内容。

处理 AI 生成更改的方式有两种：

- [**应用后审核**](/content-ai/capabilities/agent/review/ai-changes)：更改应用到文档后，通过 **AI 更改** 扩展进行审核。用户可以接受或拒绝更改。拒绝更改会将文档恢复到之前的内容。请参阅 [此指南](/content-ai/capabilities/agent/review/ai-changes) 进行配置。
- [**应用前预览**](/content-ai/capabilities/agent/review/ai-suggestion)：更改先发送至 **AI 建议** 扩展，用户可以在更改应用到文档之前进行审核并批准或拒绝。请参阅 [此指南](/content-ai/capabilities/agent/review/ai-suggestion) 进行配置。

## 自动接受配置

当 AI Agent 调用工具时，你可能希望它暂停并等待用户审核，或者继续自动运行而不需要用户交互。

你可以通过 `autoAccept` 选项配置此行为。该选项控制 AI Agent 是否应自动接受任何工具调用并继续运行，或停止等待用户审核。

- `"always"`：所有工具调用均自动应用，无需审核。AI Agent 会持续运行，直到完成任务。
- `"never"`：所有工具调用均需要用户审核和明确接受。
- `"onlyRead"`：仅修改编辑器内容的工具调用需要审核；读取操作自动接受。

```tsx
// 配置自动接受行为
const provider = new AiAgentProvider({
  reviewOptions: {
    autoAccept: 'onlyRead',
  },
  // ... 其他选项
})

// 在运行时更改自动接受行为
provider.setAutoAccept('never')
```
