所有 [编辑文档](/content-ai/capabilities/ai-toolkit/api-reference/edit-the-document) 的 AI Toolkit 方法都具有 `reviewOptions` 属性。配置该属性后，可以在文档编辑完成后显示审阅界面。

```ts
const result = toolkit.executeTool({
  toolName: 'tiptapEdit',
  input,
  // 显示一个界面，允许用户在更改应用后进行审阅。
  reviewOptions: { mode: 'review' },
})
```

编辑文档后，将显示一份[建议](/content-ai/capabilities/ai-toolkit/advanced-guides/suggestions)列表。

`reviewOptions` 属性接收一个 `ReviewOptions` 配置对象。

## `ReviewOptions`

- `mode?` （`'disabled' | 'review' | 'preview'`）：是否在应用更改前或后进行审阅。默认值：`'disabled'`
  - `'disabled'`：不显示审阅界面。
  - `'review'`：先应用更改，然后显示审阅界面，允许用户撤销更改。
  - `'preview'`：在应用更改之前显示预览界面，允许用户接受或拒绝更改。
- `displayOptions?`（`DisplayOptions`）：自定义建议的显示方式，设置 [`Suggestion`](/content-ai/capabilities/ai-toolkit/api-reference/display-suggestions#the-suggestion-object) 类型的 `displayOptions` 属性。[查看可用选项](/content-ai/capabilities/ai-toolkit/api-reference/display-suggestions#the-suggestion-object)。
- `metadata?`（`Record<string, any>`）：为建议添加的额外元数据，可用于存储附加信息（例如其来源或类别）。该属性不由扩展内部使用，但可帮助开发者自定义建议在 UI 中的显示方式。
- `useDiffUtility?`（`boolean`）：是否使用[差异工具](/content-ai/capabilities/ai-toolkit/api-reference/diff-utility)比较更改前后的文档，并以丰富的差异形式显示删除和添加的内容。若为 `false`，则删除内容和添加内容作为整块内容显示，视为单个更改。默认值：`true`
- `diffUtilityOptions?`（[`ExternalDiffUtilityOptions`](/content-ai/capabilities/ai-toolkit/api-reference/diff-utility#parameters-diffutilityoptions)）：用于差异工具比较文档的选项。[查看可用选项](/content-ai/capabilities/ai-toolkit/api-reference/diff-utility#parameters-diffutilityoptions)。
