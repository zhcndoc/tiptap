在 Tiptap，我们非常重视您的隐私。因此，在使用 Tiptap 内容 AI 时，可能会引发一些相关问题，以下我们将详细说明。

## 云端整合

Tiptap 内容 AI 云端版本 **充当您的客户端（Tiptap 编辑器）与 OpenAI 之间的代理**。这样做是为了保护您的 OpenAI 密钥不被客户端接触，因为公开共享秘密信息是一种反模式。

使用我们的云服务时，您需要在您的 AI 设置页面中输入您的 OpenAI 密钥。此密钥随后会被加密并存储在我们的数据库中。我们使用此密钥代表您向 OpenAI API 进行请求。

### 数据流

数据的总体流程如下：

1. 您的客户端（编辑器）执行诸如 `翻译` 等 AI 命令
2. AI 扩展将请求发送至我们的后台服务
3. 后台进行您的应用ID的查验，确保提供的令牌（生成的 JWT）有效，并继续请求或以错误代码中止
4. 经过授权后，后台基于所需命令构建提示符，并使用您的 OpenAI API 密钥向 OpenAI API 发送请求
5. 请求处理完成后，响应将返回给客户端

### 我们记录什么

在上述过程中，我们会定期记录以下数据，以确保服务的正常和可靠运行：

- 包含时间戳、关联ID、引用来源、用户代理和IP的HTTP日志，包括相关请求头和请求体的一部分，以保障服务正常运行并防止滥用
- 记录与您团队相关的操作，包括时间戳、关联ID、命令（如“翻译”）及使用次数，以便监控您的计划额度

为了在出现问题时能够协助您，我们还实现了一种叫做“增强日志”的机制。

**只有在明确请求（例如调试）且获得您的许可后**，我们才会启用增强日志功能。需要说明的是：这不是默认设置。然而，开启后会记录以下额外字段：

- 输入文本和参数、后台构建的提示符以及 OpenAI 的响应

这些数据帮助我们追踪各端的错误，也帮助您排查可能遇到的问题。调试完成后，我们会立即关闭增强日志。

### 加密

如前所述，我们会将以下内容在数据库中加密存储：

- 您的 OpenAI API 密钥
- 您的 JWT 密钥

这些秘密信息仅在授权或执行请求时解密。

客户端、后台以及 OpenAI 之间的所有流量都采用最新的 SSL 标准进行加密。

## 本地部署版

本地部署版本在技术上与云端版本相同，也充当客户端与 OpenAI 之间的代理。

不同之处很明显，主要包括：

- OpenAI API 密钥存储在您指定的位置，绝不传送至我们
- 我们不获得任何关于请求使用情况的统计信息
- 日志会输出到标准输出（stdout），您可以根据需要和偏好进行使用

因此，本地版本让您拥有对收集的数据和指标的完全控制权。
