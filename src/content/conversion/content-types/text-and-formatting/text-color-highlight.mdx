文本颜色和高亮会在 Word 的颜色模型与 Tiptap 的 `textStyle` 和 `highlight` 标记之间相互转换。在安装了正确的扩展后，两者都可以往返转换。

## 你需要什么

- **Extensions:** [`ConvertKit`](/conversion/import/docx/convertkit)。其内置的 `TextStyleKit` 提供 `Color`（以及 `BackgroundColor`），并且 ConvertKit 会预先注册带有 `multicolor: true` 配置的 `Highlight`。
- **Configuration:** 不需要。

```ts
import { ConvertKit } from '@tiptap-pro/extension-convert-kit'

new Editor({ extensions: [ConvertKit] })
```

<Callout title="multicolor: true 很重要" variant="warning">
  如果 `Highlight` 没有设置 `multicolor: true`，就只会支持单一的通用高亮，并且所有导入的高亮都会折叠为默认的黄色。ConvertKit 会自动应用该配置；如果你覆盖了 `highlight` 插槽，请保留这个标志。
</Callout>

## 支持概览

|                     | 导入                            | 编辑器                          | 导出                          |
| ------------------- | ------------------------------- | --------------------------------- | ----------------------------- |
| 文本颜色            | 支持（`textStyle` 标记）        | 支持（TextStyle + Color）        | 支持（规范化为十六进制）       |
| 高亮                | 支持（`highlight` 标记）        | 支持（Highlight，multicolor）    | 支持（命名颜色或十六进制）     |
| 底纹（`<w:shd>`）   | 支持（映射到 `highlight`）      | 支持                             | 支持（纯色底纹）               |
| 主题颜色            | 不支持（不会转换）              | 不适用                           | 不适用                         |

## 导入

可使用 [编辑器扩展](/conversion/import/docx/editor-extension) 或 [REST API](/conversion/import/docx/rest-api) 导入文本颜色和高亮。两者会生成完全相同的输出。

转换服务会将 Word 的颜色元素映射为 Tiptap 标记：

| Word 元素                            | Tiptap 标记 | 属性                   |
| ----------------------------------- | ----------- | ---------------------- |
| `<w:color val="FF0000">`            | `textStyle` | `{ color: "#FF0000" }` |
| `<w:highlight val="yellow">`        | `highlight` | `{ color: "yellow" }`  |
| `<w:shd val="clear" fill="00FFFF">` | `highlight` | `{ color: "#00FFFF" }` |

`<w:highlight>` 和 `<w:shd>` 都会生成相同的 `highlight` 标记。开发者无需区分它们。

<Callout title="主题颜色不会被导入" variant="warning">
  Word 主题颜色（例如 `<w:color>` 上的 `accent1`、`accent2` 或 `themeColor` 属性）在导入时不会转换。如果某个文本片段只使用了 `val="auto"` 的主题颜色，那么该颜色不会包含在输出中。只有 `<w:color val="...">` 中明确指定的十六进制颜色值会被保留。
</Callout>

## 编辑器渲染

[`Color`](/editor/extensions/functionality/color) 扩展为 [`TextStyle`](/editor/extensions/marks/text-style) 添加颜色支持，而 [`Highlight`](/editor/extensions/marks/highlight) 扩展负责处理背景颜色。ConvertKit 同时打包了这两者：`TextStyleKit` 提供 `TextStyle` + `Color` + `BackgroundColor`，并且 `Highlight` 已预先配置为 `multicolor: true`。

文本颜色会以 `<span>` 元素上的行内 `style="color: ..."` 形式渲染。高亮会以带有 `background-color` 样式的 `<mark>` 元素渲染。

如果你禁用 `ConvertKit.configure({ textStyleKit: false })`，导入的文本颜色属性将不会被 schema 识别。`highlight: false` 也是同理。有关如何处理无法识别的标记和属性，请参阅[无效 schema 指南](/guides/invalid-schema)。

## Export

You can use [Editor Extension](/conversion/export/docx/editor-extension) or [REST API](/conversion/export/docx/rest-api) to export text color and highlight. Both handle color and highlight in exactly the same way.

When exporting, color values are normalized to 6-digit hexadecimal. The following input formats are supported:

- Hex with a hash: `#FF0000`, `#F00`, `#FF0000AA`
- RGB/RGBA: `rgb(255, 0, 0)`, `rgba(255, 0, 0, 0.5)`
- Named CSS colors: `red`, `blue`, `green`, `yellow`, `cyan`, `purple`, and 14 others

Highlights are exported as named Word highlight colors (when the color matches a Word enum value, such as `yellow` or `cyan`), or as solid character shading with a hexadecimal fill value. If the `highlight` mark has no color attribute, yellow is used by default.
