`@tiptap-pro/extension-export-doc` 扩展内置支持定制导出文档的页眉和页脚。您可以为首页、奇数页和偶数页配置不同的页眉和页脚。

## 页眉配置

`headers` 对象允许您自定义导出 DOC 文档的页眉：

| 属性                | 类型         | 描述                                                                                                  |
| ------------------- | ------------ | ---------------------------------------------------------------------------------------------------- |
| `evenAndOddHeaders` | `boolean`    | 是否对奇数页和偶数页使用不同的页眉                                                                     |
| `differentFirstPage` | `boolean`    | 是否首页使用不同的页眉。为 `true` 时，首页使用 `first` 值代替 `default`。                                    |
| `default`           | `string`     | 默认页眉，应用于所有页面，或当 `evenAndOddHeaders` 生效时应用于奇数页。支持纯文本字符串或序列化的 Tiptap JSONContent，用于丰富格式化。 |
| `first`             | `string`     | 首页的页眉，仅当 `differentFirstPage` 为 `true` 时生效。支持纯文本字符串或序列化的 Tiptap JSONContent。                 |
| `even`              | `string`     | 当启用 `evenAndOddHeaders` 时，偶数页的页眉。支持纯文本字符串或序列化的 Tiptap JSONContent。                     |

使用纯文本时，会生成简单无样式的页眉。使用序列化的 Tiptap JSONContent（通过 `JSON.stringify()`）时，则会保留加粗、斜体和链接等丰富格式。

## 页脚配置

`footers` 对象允许您自定义导出 DOC 文档的页脚：

| 属性                | 类型         | 描述                                                                                                  |
| ------------------- | ------------ | ---------------------------------------------------------------------------------------------------- |
| `evenAndOddFooters` | `boolean`    | 是否对奇数页和偶数页使用不同的页脚                                                                     |
| `differentFirstPage` | `boolean`    | 是否首页使用不同的页脚。为 `true` 时，首页使用 `first` 值代替 `default`。                                    |
| `default`           | `string`     | 默认页脚，应用于所有页面，或当 `evenAndOddFooters` 生效时应用于奇数页。支持纯文本字符串或序列化的 Tiptap JSONContent。 |
| `first`             | `string`     | 首页的页脚，仅当 `differentFirstPage` 为 `true` 时生效。支持纯文本字符串或序列化的 Tiptap JSONContent。                 |
| `even`              | `string`     | 当启用 `evenAndOddFooters` 时，偶数页的页脚。支持纯文本字符串或序列化的 Tiptap JSONContent。                     |

使用纯文本时，会生成简单无样式的页脚。使用序列化的 Tiptap JSONContent（通过 `JSON.stringify()`）时，则会保留加粗、斜体和链接等丰富格式。

## 完整示例

```js
import { ExportDoc } from '@tiptap-pro/extension-export-doc'

const editor = new Editor({
  extensions: [
    // 其他扩展...
    ExportDoc.configure({
      token: 'YOUR_TOKEN',
      appId: 'YOUR_APP_ID',
      headers: {
        evenAndOddHeaders: true,
        default: '我的文档 - 机密',
        first: '欢迎阅读我的文档',
        even: '我的文档 - 偶数页',
      },
      footers: {
        evenAndOddFooters: true,
        default: '公司名称 - 版权所有',
        first: '草稿版本 1.0',
        even: '公司名称 - 偶数页页脚',
      },
    }),
    // 其他扩展...
  ],
})

// 带页眉和页脚导出
editor
  .chain()
  .exportDoc({
    onCompleteExport(result) {
      const url = URL.createObjectURL(result)
      const a = document.createElement('a')

      a.href = url
      a.download = 'document-with-headers.doc'
      a.click()

      URL.revokeObjectURL(url)
    },
  })
  .run()
```

## 简单的页眉和页脚

如果您不需要奇偶页不同的页眉，可以只提供 `default` 的值：

```js
ExportDoc.configure({
  token: 'YOUR_TOKEN',
  appId: 'YOUR_APP_ID',
  headers: {
    default: '我的文档标题',
  },
  footers: {
    default: '页脚 - 公司名称',
  },
})
```
