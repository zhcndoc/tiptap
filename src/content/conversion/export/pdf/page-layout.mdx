The PDF export extension supports custom page sizes and margins, allowing you to create documents with precise layouts that meet your needs. Whether you need A5 paper, a custom paper size, or a specific margin configuration, you can set it directly in the extension.

## 页面大小配置

Use the `pageSize` option to define custom page dimensions for the exported PDF file. Page size configuration supports width and height values in multiple units.

### 支持的单位

All page size and margin dimensions support the following units:

| 单位 | 说明                 |
| ---- | -------------------- |
| `cm` | Centimeters (default) |
| `in` | Inches                |
| `pt` | Points                |
| `pc` | Picas                 |
| `mm` | Millimeters           |
| `px` | Pixels                |

### 配置选项

| 属性     | 类型     | 说明                                                                                          | 默认值     |
| -------- | -------- | --------------------------------------------------------------------------------------------- | ---------- |
| `width`  | `string` | Page width. Must be a positive number followed by a valid unit (cm, in, pt, pc, mm, px).      | `"21cm"`   |
| `height` | `string` | Page height. Must be a positive number followed by a valid unit (cm, in, pt, pc, mm, px).     | `"29.7cm"` |

```js
import { ExportPdf } from '@tiptap-pro/extension-export-pdf'

ExportPdf.configure({
  token: 'YOUR_TOKEN',
  appId: 'YOUR_APP_ID',
  pageSize: {
    width: '14.8cm', // A5 width
    height: '21cm', // A5 height
  },
})
```

## 页面边距配置

The `pageMargins` option allows you to set custom margins for each side of the document page. Unlike page size, the top and bottom margins can accept negative values.

### 配置选项

| 属性      | 类型     | 说明                                                                                                                                  | 默认值      |
| --------- | -------- | ------------------------------------------------------------------------------------------------------------------------------------- | ----------- |
| `top`     | `string` | Top margin of the page. Can be negative. Must be a number followed by a valid unit (cm, in, pt, pc, mm, px).                        | `"1cm"`    |
| `bottom`  | `string` | Bottom margin of the page. Can be negative. Must be a number followed by a valid unit (cm, in, pt, pc, mm, px).                     | `"1cm"`    |
| `left`    | `string` | Left margin of the page. Must be a positive number followed by a valid unit (cm, in, pt, pc, mm, px).                              | `"1cm"`    |
| `right`   | `string` | Right margin of the page. Must be a positive number followed by a valid unit (cm, in, pt, pc, mm, px).                             | `"1cm"`    |
| `header`  | `string` | Distance from the top edge of the page to the top of the header. Should be less than `top`. Must be a positive number with a unit. | `"1.25cm"` |
| `footer`  | `string` | Distance from the bottom edge of the page to the bottom of the footer. Should be less than `bottom`. Must be a positive number with a unit. | `"1.25cm"` |

```js
import { ExportPdf } from '@tiptap-pro/extension-export-pdf'

ExportPdf.configure({
  token: 'YOUR_TOKEN',
  appId: 'YOUR_APP_ID',
  pageMargins: {
    top: '2cm',
    bottom: '2cm',
    left: '1.5cm',
    right: '1.5cm',
    // Distance from page edge to header/footer
    header: '1cm',
    footer: '1cm',
  },
})
```

<Callout title="Automatically derived from Pages extension" variant="hint">
  When the [Pages extension](/editor/extensions/functionality/pages) is configured together with `ExportPdf`, the `header` and `footer` offsets are automatically derived from the `headerTopMargin` and `footerBottomMargin` values stored in Pages. The values you pass to `pageMargins` always take precedence.
</Callout>

<Callout title="Conversion service requirements" variant="info">
  End-to-end support for `pageMargins.header` and `pageMargins.footer` requires the Tiptap convert service to use a version that supports these fields and provide the updated `@tiptap-pro/extension-export-docx`. In older convert service versions, these values are ignored and Word defaults are used.
</Callout>

## 页面背景颜色

Use the `pageColor` option to set a background color for every page of the exported PDF.

### 接受的格式

| 格式          | 示例                                                     |
| ------------- | -------------------------------------------------------- |
| Hex（6 位）   | `"#ff0000"`                                             |
| Hex（3 位）   | `"#f00"`                                                |
| `rgb()`       | `"rgb(255, 0, 0)"`                                      |
| `rgba()`      | `"rgba(255, 0, 0, 0.5)"`                                |
| 命名颜色      | 常见的 CSS 命名颜色（`"red"`、`"blue"`、`"green"` 等） |

传入 `"transparent"`，或将 `pageColor` 保持未定义，即可省略页面颜色。

```js
ExportPdf.configure({
  token: 'YOUR_TOKEN',
  appId: 'YOUR_APP_ID',
  pageColor: '#fff8dc',
})
```

## Complete example

以下是一个完整示例，展示如何使用 PDF 导出扩展配置自定义页面布局：

```js
import { ExportPdf } from '@tiptap-pro/extension-export-pdf'

const editor = new Editor({
  extensions: [
    // 其他扩展...
    ExportPdf.configure({
      token: 'YOUR_TOKEN',
      appId: 'YOUR_APP_ID',
      // A5 页面尺寸
      pageSize: {
        width: '14.8cm',
        height: '21cm',
      },
      // 自定义边距
      pageMargins: {
        top: '2cm',
        bottom: '2cm',
        left: '2cm',
        right: '2cm',
      },
      // 页面背景颜色
      pageColor: '#fff8dc',
    }),
    // 其他扩展...
  ],
})

// 使用自定义布局导出
editor
  .chain()
  .exportPdf({
    onCompleteExport(result) {
      const url = URL.createObjectURL(result)
      const a = document.createElement('a')

      a.href = url
      a.download = 'custom-layout.pdf'
      a.click()

      URL.revokeObjectURL(url)
    },
  })
  .run()
```

## 常用页面尺寸

以下是一些常用的页面尺寸供参考：

| 格式        | 宽度     | 高度     |
| ----------- | -------- | -------- |
| **A4**      | `21cm`   | `29.7cm` |
| **A5**      | `14.8cm` | `21cm`   |
| **Letter**  | `8.5in`  | `11in`   |
| **Legal**   | `8.5in`  | `14in`   |
| **Tabloid** | `11in`   | `17in`   |

## 不同计量单位混用

您可以根据喜好混合使用不同的单位：

```js
ExportPdf.configure({
  token: 'YOUR_TOKEN',
  appId: 'YOUR_APP_ID',
  // 美制 Letter 尺寸，单位为英寸
  pageSize: {
    width: '8.5in',
    height: '11in',
  },
  // 使用不同单位的边距
  pageMargins: {
    top: '0.75in', // 英寸
    bottom: '72pt', // 磅（1 英寸 = 72 磅）
    left: '2cm', // 厘米
    right: '20mm', // 毫米
  },
})
```

<Callout title="单位一致性" variant="hint">
  虽然您可以混用不同单位，但通常建议在整个配置中使用一致的单位，以提高可维护性和清晰度。
</Callout>
