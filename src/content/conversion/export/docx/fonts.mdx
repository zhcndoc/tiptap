Word 只能渲染读者设备上已安装的字体——除非字体本身被 **嵌入** 到文档中。`@tiptap-pro/extension-export-docx` 可以将内容所使用的字体嵌入进去，因此导出的 `.docx` 在任何地方看起来都一样，即使那些字体没有安装也是如此。

有两种方式可以实现：

- **[手动](#manual-embedding)** — 你通过 `fonts` 选项提供字体字节。完全可控，无需网络依赖，可在服务器上工作。
- **[自动](#automatic-embedding)** — 设置 `embedFonts: true`，导出时会自动检测、获取并嵌入文档所使用的字体。仅限浏览器。

<Callout title="仅嵌入常规变体">
  DOCX 字体嵌入会携带每个字族的常规（直立、400 字重）字形。Word 会基于它合成粗体和斜体——你不需要（也不能通过此 API）嵌入单独的粗体/斜体字体文件。
</Callout>

## 手动嵌入

传入一个 `fonts` 数组——每种字体字族一个条目，每个条目都包含原始的 TTF 或 OTF 字节。字体会被混淆后写入文档包中（按 OOXML 规范为 `word/fonts/font<N>.odttf`），并绑定到每个使用匹配字族名称的文本片段。

```ts
import { ExportDocx } from '@tiptap-pro/extension-export-docx'

// 在浏览器中：获取字体并将其 ArrayBuffer 直接交给导出。
const data = await fetch('/fonts/PlayfairDisplay-Regular.ttf').then((response) =>
  response.arrayBuffer(),
)

editor
  .chain()
  .exportDocx({
    fonts: [{ name: 'Playfair Display', data }],
  })
  .run()
```

在服务器上，则改为将文件读入 `Buffer`：

```ts
import { readFile } from 'node:fs/promises'
import { exportDocx } from '@tiptap-pro/extension-export-docx'

const data = await readFile('./fonts/PlayfairDisplay-Regular.ttf')

await exportDocx({
  document: editorJSON,
  exportType: 'buffer',
  customNodes: [],
  styleOverrides: {},
  fonts: [{ name: 'Playfair Display', data }],
})
```

### `DocxFontDefinition`

```ts
interface DocxFontDefinition {
  name: string
  data: Buffer | Uint8Array | ArrayBuffer
  characterSet?: DocxFontCharacterSet
}
```

| 字段           | 描述                                                                                                                                                                                                                                                                                                                                     |
| -------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `name`         | 字体家族名称。**必须与**内容中使用的字体名称完全匹配——即 `textStyle` 标记的 `fontFamily` 的第一个家族（去掉引号，因此 `'"Playfair Display", serif'` 会解析为 `Playfair Display`），或者通过 `styleOverrides` / `textRunOverrides` 设置的 `font`。不匹配时会静默回退到替代字体。 |
| `data`         | 原始的 **TTF 或 OTF** 字节。接受 Node `Buffer`、`Uint8Array` 或 `ArrayBuffer`（因此 `await fetch(url).then((r) => r.arrayBuffer())` 可在浏览器中直接使用）。这里不接受 WOFF/WOFF2 数据——请参见 [自动嵌入](#automatic-embedding) 了解 WOFF2 转换。                                                      |
| `characterSet` | 可选的 Word 字符集代码，写入字体表中的 `<w:charset>`。普通拉丁字体可留空。请使用重新导出的 `CharacterSet` 常量作为该值（例如 `CharacterSet.GREEK`）。                                                                                                                                       |

## 自动嵌入

设置 `embedFonts: true` 后，导出会自动完成工作：它会收集文档引用到的每个字体家族，定位每个字体文件，必要时将其转换为可嵌入格式，并将其嵌入——无需为每次导出单独处理字体。

```ts
ExportDocx.configure({
  embedFonts: true,
  // 仅当字体需要进行 WOFF2 → TTF 转换时才会使用 appId + token
  appId: 'your-app-id',
  token: 'your-jwt',
  endpoint: 'https://api.tiptap.dev/v2/convert', // 可选，这是默认值
  onCompleteExport: (blob) => {
    /* 下载 blob */
  },
})
```

### 工作原理

1. **检测** — 收集在 `textStyle` 的 `fontFamily` 标记、[CSS 样式](/conversion/export/docx/css-to-docx)以及运行级样式覆盖中使用的字族。通用关键字（`serif`、`monospace` 等）以及你已经通过 `fonts` 传入的字族会被跳过。
2. **定位** — 从页面可读的 `@font-face` 规则中查找每个字体文件，选择常规字形。当某个字族按语言拆分成多个子集时——例如 Google Fonts 以及许多自托管方案——会选择覆盖文档实际使用字符的那个子集：拉丁文档嵌入拉丁字形，西里尔文档嵌入西里尔字形，等等。没有可读规则的字族——通常是 Google Fonts 的 `<link>` 情况，因为其跨域样式表无法被 JavaScript 读取——会回退到 [Google Fonts](https://fonts.google.com) 查询。
3. **转换** — `TTF`/`OTF` 文件会按原样嵌入。`WOFF2` 文件会通过 Convert Service 的 `POST /fonts/convert` 端点转换为 TTF（这一步需要 `appId` / `token`）。
4. **嵌入** — 最终字体会像你通过手动 `fonts` 选项传入一样被嵌入。手动提供的字体总是优先于自动解析到的同名字体。

### 要求和凭证

- **仅限浏览器。** 自动嵌入会读取 `document.styleSheets`，因此在服务器环境中不会执行任何操作。服务器端请使用手动 `fonts` 选项。
- **仅当需要 WOFF2 转换时才需要 `appId` / `token`。** 自托管的 `TTF`/`OTF` 字体无需调用 Convert Service 即可嵌入。WOFF2 字体（包括所有从 Google Fonts 解析得到的字体）需要这些凭证，以便 [转换端点](#convert-service-endpoint) 运行。
- **Convert Service ≥ v2.25.0** 必须作为你的 `endpoint` 后端，才能使用 WOFF2 → TTF 转换。

<Callout title="嵌入绝不会破坏导出">
  每个字体都会被独立解析。如果某个字体无法找到、获取或转换——或者 WOFF2 字体缺少凭证——导出只会为该字族记录一条 `console.warn`，然后继续。受影响的文本会回退到 Word 的字体替换机制，完全与未嵌入时相同；导出本身始终会成功。
</Callout>

<Callout title="一个字体用于多种脚本">
  一个 DOCX 会为每个字体家族嵌入一个文件。如果在单个文档中，**同一**字体家族被用于多种脚本的文本（例如拉丁文和西里尔文），则会嵌入覆盖该文本最多部分的子集，而其他脚本的文本会回退到 Word 的替换机制。若要为此类字体家族嵌入完整覆盖范围，请提供一个完整的（非子集）字体文件——将其作为单个 `@font-face` 自托管，或者通过手动 [`fonts`](#manual-embedding) 选项传入。仅用于单一脚本的字体家族不受影响。
</Callout>

### 选项

| 参数          | 描述                                                                                                     | 默认值                              |
| ------------ | --------------------------------------------------------------------------------------------------------------- | ----------------------------------- |
| `embedFonts` | 启用自动检测和嵌入。仅限浏览器。                                                         | `false`                             |
| `appId`      | Tiptap Convert 应用 ID，作为 `X-App-Id` 标头发送。仅在 WOFF2 字体需要转换时必需。         | `undefined`                         |
| `token`      | Tiptap Convert JWT，作为 `Authorization` bearer token 发送。仅在 WOFF2 字体需要转换时必需。 | `undefined`                         |
| `endpoint`   | 用于 WOFF2 → TTF 转换的 Tiptap Convert REST 端点基础地址。                                              | `https://api.tiptap.dev/v2/convert` |

## 让编辑器与导出结果一致

为了让编辑器预览与嵌入后的输出一致，请在页面中使用 `@font-face` 规则加载相同字体（这也是自动嵌入读取以定位自托管字体的方式）：

```css
@font-face {
  font-family: 'Playfair Display';
  src: url('/fonts/PlayfairDisplay-Regular.ttf') format('truetype');
  font-weight: 400 900;
  font-style: normal;
  font-display: swap;
}
```

然后使用 `fontFamily` 标记将该字体应用到内容上——其清理后的名称就是编辑器和导出都会绑定的名称：

```ts
editor.chain().focus().setFontFamily('Playfair Display').run()
```

## Convert Service 端点

自动嵌入中的 WOFF2 转换由一个 REST 端点提供支持，你也可以直接调用它：

`POST {endpoint}/fonts/convert`

- **Body:** 带有 `file` 字段（字体二进制）以及可选 `fontFamily` 字段的 `multipart/form-data`。
- **Response:** 可嵌入的字体，类型为 `font/ttf`——TTF/OTF 上传会原样返回，WOFF2 上传会转换为 TTF。
- **Auth:** 与其他转换端点相同的 `Authorization: Bearer <token>` 和 `X-App-Id` 标头。

可在 Tiptap Cloud 和本地部署环境中使用，前提是 Convert Service v2.25.0。
