Tiptap 离不开其活跃的社区。贡献一直以来都是被欢迎的，未来也将继续欢迎。以下是您在提交贡献之前需要了解的一些事项：

## 欢迎示例

- 作为 Bug 报告的回归测试失败
- 文档改进，例如修复拼写错误，添加某个部分
- 为现有扩展添加新功能，例如新的可配置选项
- 对核心的解释清晰、不破坏现有功能的变更

## 不会合并

- 新扩展，这将需要我们支持和维护

## 提交想法

请确保先打开一个问题并概述您的想法。我们会迅速回复您，让您知道是否有机会合并您的贡献。

## 设置开发环境

对官方仓库进行修改并不难。您需要安装 [Git](https://github.com/git-guides/install-git)、[Node 和 NPM](https://nodejs.org/en/download/)。接下来您需要做的是：

1. 将代码复制到本地计算机：`$ git clone git@github.com:ueberdosis/tiptap.git`
2. 安装依赖：`$ npm install`
3. 启动开发环境：`$ npm run start`
4. 在您喜欢的浏览器中打开 http://localhost:3000。
5. 开始玩耍吧！

## 我们的代码风格

有一个 eslint 配置来确保一致的代码风格。要检查错误，请运行 `$ npm run lint`。在您发送拉取请求时也会进行检查。在发送拉取请求之前，确保代码通过检查。

## 检查错误

您的拉取请求会自动执行我们现有的所有测试。在提交拉取请求之前，请确保它们全部通过。可以使用 `$ npm run test` 在本地运行所有测试，或使用 `$ npm run test:open` 运行单个测试（例如，当编写新测试时）。

## 创建自己的扩展

如果您想创建和维护自己的扩展，可以使用 `create-tiptap-extension` CLI 工具。它会创建一个包含所有必要文件和构建过程的新扩展模板。只需运行

```bash
npm init tiptap-extension
```

如果您想让我们知道您的扩展，可以在 [X](https://x.com/tiptap_editor) 或 [Discord](https://tiptap.dev/discord) 上给我们提示。

## 记录您的贡献

在贡献 Tiptap 时，重要的是要理解 Tiptap 的代码库和文档是分别管理的。如果您对 Tiptap 进行了更改或增强，记录这些更改对于他人的清晰性和可用性当然是重要的。确保您更新与您在代码中所做的任何更改相对应的文档仓库。

1. Tiptap 仓库：[Tiptap 代码仓库](https://github.com/ueberdosis/tiptap) - 在此修改代码。
2. 文档仓库：[Tiptap 文档仓库](https://github.com/ueberdosis/tiptap-docs) - 在此更新或添加文档，以反映在 Tiptap 仓库中所做的更改。

在您对 Tiptap 代码仓库的拉取请求中，包含对您在文档仓库中所做的相应更新的引用。或者，在向文档仓库提交拉取请求时，如果与文档更新相关，确保包含对 Tiptap 仓库中任何相关代码更改的引用。
